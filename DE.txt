Data Formats and Storage — Syllabus
Goal: Understand how different data formats are stored and used efficiently in data pipelines.

1. Data Storage Fundamentals
Topics:
•	Why data formats matter in data engineering
•	Structured vs semi-structured data
•	Row-based vs column-based storage
•	Trade-offs between storage formats
Concepts:
•	storage efficiency
•	query performance
•	compression
•	schema structure
Outcome: Understanding how storage format affects pipeline performance and analytics efficiency.

2. CSV (Comma-Separated Values)
Overview:
Simple text-based tabular data format.
Topics:
•	CSV file structure
•	delimiters
•	headers
•	encoding formats
Operations:
•	reading CSV files
•	writing CSV files
•	handling large CSV files
•	dealing with missing values
Characteristics:
Advantages:
•	easy to create
•	human readable
•	widely supported
Limitations:
•	large file sizes
•	slower processing
•	no schema enforcement
Outcome: Ability to use CSV for simple data ingestion and exchange.

3. JSON (JavaScript Object Notation)
Overview:
Common format for semi-structured data.
Topics:
•	JSON structure
•	nested objects
•	arrays
•	key-value representation
Operations:
•	reading JSON data
•	parsing nested JSON
•	flattening JSON structures
Use cases:
•	API responses
•	event logs
•	configuration data
Limitations:
•	larger storage size
•	slower analytical queries
Outcome: Ability to handle semi-structured data formats commonly used in APIs and event systems.

4. Parquet (Columnar Storage Format)
Overview:
Optimized storage format used in modern data pipelines.
Topics:
•	columnar storage structure
•	compression mechanisms
•	efficient column reading
•	schema support
Operations:
•	reading Parquet files
•	writing Parquet datasets
•	partitioned Parquet storage
Advantages:
•	high compression
•	fast analytical queries
•	efficient column selection
Common usage:
•	data lakes
•	big data processing
•	data warehouse storage
Outcome: Ability to store large datasets efficiently using columnar formats.

5. Columnar Storage Basics
Concept: Data stored by columns instead of rows.
Row-based storage:
Each row stored sequentially.
Column-based storage:
Each column stored together.
Benefits:
•	faster analytical queries
•	reduced disk I/O
•	better compression
Example scenario: Querying one column in columnar storage reads only that column instead of entire rows.
Outcome: Understanding why columnar formats are preferred for analytics workloads.

6. Choosing the Right Format
Guidelines:
CSV:
•	small datasets
•	simple data exchange
JSON:
•	semi-structured data
•	API responses
•	event logs
Parquet:
•	large datasets
•	data lakes
•	analytics systems
Outcome: Ability to select appropriate storage formats for different pipeline stages.
